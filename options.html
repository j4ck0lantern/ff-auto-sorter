Hobby Sorter Options

Configure your hobbies below. The configuration must be a JSON Array (starts with [ ... ]).

The array contains objects, and bookmarks are checked in the order you define them. This gives you full control over priority.

Each object in the array must have two keys:

  • "folder": A string defining the folder path. Use / to create subfolders (e.g., "Programming/Web").
  • "config": An object containing your matching rules:
    • "keywords": (Optional) A list of simple string keywords.
    • "comment": (Optional) A string to add notes for yourself.
    • "regex": (Optional) A list of objects, where each object has:
      • "pattern": The regex string. (Remember to escape backslashes: "\\.pdf$").
      • "comment": A string to explain what this regex does.

Example of one item in the array:

[ { "folder": "Programming/Web", "config": { "keywords": ["react", "css", "node"], "regex": [ { "pattern": "github\\.com\\/[^\\/]+\\/(js|react)-", "comment": "Matches GitHub repos with 'js-' or 'react-' in the name" } ], "comment": "Web dev links. This will be checked first." } }, { "folder": "Programming", "config": { "keywords": ["github", "stackoverflow"], "comment": "Catch-all for other programming links. Checked second." } } ]

A bookmark will be filed under the first rule that matches.
